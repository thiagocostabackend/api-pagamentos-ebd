# Gerenciador de Pagamentos – EBD

Este projeto nasceu a partir de uma situação real vivenciada durante minha passagem como **tira-férias da secretaria da Escola Bíblica Dominical (EBD)**.

Na rotina administrativa, o controle dos pagamentos das revistas adquiridas pelos membros matriculados era feito de forma manual, utilizando papel, caneta e memória, o que acabava gerando inconsistências, retrabalho e erros no fechamento dos valores.

Após concluir uma capacitação em **Back-End com Python**, decidi aplicar de forma simples e objetiva os conhecimentos adquiridos, transformando esse problema real em uma **API backend** para organização e controle desses pagamentos.

---

## Objetivo do Projeto

Criar uma API simples para:

- Gerenciar membros matriculados na EBD  
- Registrar pagamentos de revistas  
- Controlar formas de pagamento (PIX e espécie)  
- Facilitar a consulta e conferência de valores pagos  

O foco do projeto é **aprendizado prático**, clareza de código e aplicação de boas práticas, sem complexidade desnecessária.

---

## Tecnologias Utilizadas

- Python  
- MySQL  
- API REST  
- Padrão de Projeto DAO  

> Este projeto **não utiliza Django** propositalmente. A ideia é manter a estrutura enxuta e focada em fundamentos de backend.  
> O Django será aplicado em um projeto futuro que dará continuidade a este contexto.

---

## Decisões de Projeto

- Utilização do **Padrão DAO** para separar o acesso ao banco de dados da lógica da aplicação  
- Organização em camadas para facilitar manutenção e evolução  
- Desenvolvimento incremental, baseado em necessidades reais observadas na prática  

Os requisitos não passaram por uma fase formal de levantamento, pois foram identificados diretamente durante a vivência na secretaria da EBD.

---

## Status do Projeto

Em desenvolvimento.

Este projeto representa o **primeiro braço** de um sistema maior que poderá evoluir para um **Sistema de Gerenciamento de Igrejas**, a partir de feedbacks positivos e validação prática da solução.
